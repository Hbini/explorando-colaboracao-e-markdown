# Guia de Contribuição

Primeiramente, muito obrigado por considerar contribuir com este projeto! É pessoas como você que tornam este projeto uma ótima referência.

## Como Contribuir

### Reportando Bugs

Antes de criar um relatório de bug, verifique se o problema já foi reportado. Quando você cria um relatório de bug, inclua:

* **Use um título descritivo** para identificar o problema
* **Descreva os passos exatos** que reproduzem o problema
* **Descreva o comportamento observado** e aponte o que exatamente é o problema

### Sugerindo Melhorias

As sugestões de melhorias são bem-vindas! Ao criar um relatório de melhoria, inclua:

* **Use um título descritivo** para a sugestão
* **Forneça uma descrição detalhada** da melhoria sugerida

### Pull Requests

* Mantenha os pull requests focados em um único assunto
* Escreva uma boa mensagem de commit
* Siga as convenções Conventional Commits

## Processo de Desenvolvimento

1. Fork o repositório e crie sua branch a partir de `main`
2. Faça suas mudanças e se certifique que elas funcionam
3. Commit suas mudanças com mensagens significativas
4. Push para a branch
5. Abra um Pull Request com uma descrição clara do que foi mudado

## Padrão de Commits

Usamos o padrão Conventional Commits:

* feat: para uma nova feature
* fix: para um bug fix
* docs: para mudanças de documentação
* refactor: para refatoração de código
* test: para adição de testes
